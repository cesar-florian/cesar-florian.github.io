Infraestructura web segura

Problema

Desplegar infraestructura web en la nube requiere conocimientos especializados en redes, seguridad, sistemas operativos y arquitectura cloud. Las organizaciones enfrentan desafíos al crear ambientes seguros y escalables de forma manual, ya que es complejo, propenso a errores y consume tiempo valioso del equipo técnico. Estos desafíos incluyen protección contra ataques cibernéticos, gestión de tráfico geográfico, escalabilidad automática, certificados SSL/TLS, alta disponibilidad multi-zona y control de costos operativos.

Resumen de la solución

  • Objetivo: Desplegar una infraestructura web completa, segura y escalable en Oracle Cloud Infrastructure utilizando infraestructura como código, con protección WAF avanzada, balanceo de carga, autoscaling, certificados SSL automatizados y monitoreo centralizado.

  • Automatización IaC: Terraform con módulos reutilizables por servicio, variables configurables por ambiente, outputs para integración CI/CD, state management para idempotencia, y versionado en Git para control de cambios.

  • Valor para el negocio: Reducción del 80% en tiempo de despliegue de infraestructura, protección automática contra amenazas web comunes, escalabilidad automática que optimiza costos (pagar solo por lo que se usa), alta disponibilidad del 99.99%, certificados SSL y cumplimiento de mejores prácticas de seguridad en la nube.

Público principal

  • Equipos DevOps y SRE que buscan automatizar infraestructura cloud
  • Desarrolladores que necesitan entornos seguros para aplicaciones web
  • Arquitectos de soluciones de nube evaluando Oracle Cloud Infrastructure
  • Startups y empresas que requieren infraestructura escalable con presupuesto optimizado y/o que necesitan cumplir con estándares de seguridad

Objetivos

  1. Automatizar completamente el despliegue de infraestructura web mediante Terraform
  2. Implementar seguridad en múltiples capas (red, aplicación, datos)
  3. Garantizar alta disponibilidad mediante distribución multi-zona
  4. Habilitar escalabilidad automática basada en métricas de CPU
  5. Proporcionar certificados SSL/TLS con renovación automática
  6. Centralizar logs y métricas para observabilidad completa
  7. Garantizar calidad de código mediante testing automatizado y validaciones
  8. Soportar múltiples sistemas operativos (Ubuntu y Windows Server)

Arquitectura de red

La solución implementa una arquitectura de red segmentada con los siguientes componentes:

VCN (Virtual Cloud Network): Red virtual aislada con CIDR 10.0.0.0/16

Subredes:

  • Subred pública: CIDR 10.0.1.0/24 que aloja el Load Balancer con acceso directo a Internet
  • Subred privada: CIDR 10.0.2.0/24 contiene las instancias de aplicación sin IPs públicas

Gateways:

  • Internet Gateway (IGW): Permite tráfico saliente/entrante desde subred pública
  • NAT Gateway: Habilita tráfico saliente desde subred privada hacia Internet
  • Service Gateway: Acceso privado a servicios de Oracle (Object Storage, etc.)

Tablas de Rutas:

  • Ruta Pública:
    • 0.0.0.0/0 → Internet Gateway
  • Ruta Privada:
    • 0.0.0.0/0 → NAT Gateway
    • Oracle Services → Service Gateway

Security Lists:

  • Subred pública:
    • Permite HTTP (80) desde Internet
    • Permite HTTPS (443) desde Internet
  • Subred privada:
    • Permite HTTP (80) desde 10.0.0.0/16
    • Permite HTTPS (443) desde 10.0.0.0/16
    • Permite SSH (22) desde 10.0.0.0/16
    • Permite RDP (3389) desde 10.0.0.0/16
    • Permite tráfico desde Load Balancer y Bastion

Componentes y servicios

Componente OCI Función en el proyecto
VCN (Virtual Cloud Network) Red virtual aislada que contiene toda la infraestructura
Load Balancer (Flexible) Distribuye tráfico HTTP/HTTPS entre instancias backend, health checks

Instance Pool Grupo de instancias compute con configuración idéntica
Autoscaling Configuration Escala automáticamente instancias según CPU (p. ej. 70% up, 30% down)
Compute Instances VMs Ubuntu o Windows Server ejecutando aplicación web
DNS Zone Gestión de registros DNS para el dominio
Certificates Certificados SSL/TLS vía ACME
Logging Centraliza logs de Load Balancer, WAF y auditoría
Bastion Service Acceso seguro SSH/RDP a instancias privadas sin IPs públicas
Internet Gateway Conectividad pública para subred pública
NAT Gateway Salida a Internet para instancias privadas
Service Gateway Acceso privado a servicios Oracle sin atravesar Internet

Decisiones técnicas

Decisión Opción elegida Alternativas descartadas Justificación
Proveedor Cloud Oracle Cloud Infrastructure (OCI) AWS, Azure, GCP OCI ofrece tier "Always Free" generoso (4 OCPUs Ampere A1, 24GB RAM, 200GB storage), Load Balancer flexible gratuito, y costos competitivos para cargas de trabajo adicionales
IaC Tool Terraform OCI Resource Manager Terraform es cloud-agnostic, tiene provider OCI maduro, sintaxis declarativa clara, state management robusto, y amplia comunidad. Resource Manager está limitado a OCI únicamente
Load Balancer OCI Flexible Load Balancer OCI API Gateway, Network Load Balancer, instancia NGINX Flexible LB incluido en Always Free, soporta HTTP/HTTPS, health checks nativos, integración con WAF, SSL offloading
WAF OCI Web Application Firewall CloudFlare, AWS WAF Integración nativa con OCI LB, reglas OWASP preconfiguradas, geo-blocking incorporado, rate limiting, sin costo adicional en Always Free
Certificados SSL Let's Encrypt vía ACME + DNS Challenge Certificados autofirmados, comprados Let's Encrypt es gratuito, confiable, renovación automática, DNS challenge permite validación sin exponer HTTP, ampliamente aceptado
DNS Provider OCI DNS Service CloudFlare, ZeroSSL Integración nativa con OCI, API para ACME challenge, gestión centralizada
Logging OCI Logging Servicio nativo OCI, integración directa con LB/WAF, retención configurable, búsqueda incorporada, incluido en Always Free
Acceso Remoto OCI Bastion Service VPN, acceso directo SSH Bastion managed elimina gestión de VMs adicionales, auditoría integrada, sesiones temporales, sin costo adicional
Multi-AZ Distribución en 3 Availability Domains Single AZ, multi-región Alta disponibilidad sin complejidad de multi-región, OCI tiene 3 ADs en regiones principales, failover automático
Módulos Terraform Modularización por servicio Monolítico, por ambiente Reutilización de código, testing independiente, mantenibilidad, separación de responsabilidades

Aspectos de seguridad

Capa de red

  • Segmentación pública/privada
  • Security Lists restrictivas (principio de mínimo privilegio)
  • NAT Gateway para salida controlada
  • Sin IPs públicas en instancias de aplicación

Capa de aplicación (WAF)

  • XSS Protection: Bloquea scripts maliciosos en inputs
  • SQL Injection Protection: Detecta patrones de inyección SQL
  • OWASP Top 10: Reglas contra vulnerabilidades comunes
  • RCE Protection: Previene ejecución remota de código (Unix/Windows)
  • RFI/LFI Protection: Bloquea inclusión de archivos remotos/locales
  • Geo-blocking: Restringe acceso por país (configurable)
  • Rate Limiting: Limita requests por IP (configurable)

Capa de transporte

  • TLS 1.2 y 1.3 únicamente
  • Certificados Let's Encrypt confiables
  • SSL offloading en Load Balancer

Capa de acceso

  • Bastion Service para acceso administrativo
  • Sesiones temporales (3 horas máximo)
  • Auditoría de conexiones
  • SSH keys para autenticación (sin passwords)

Capa de datos

  • Logs encriptados en reposo
  • Secrets sensibles marcados como sensitive = true en Terraform
  • API keys almacenadas localmente (~/.oci/)

Consideraciones

Limitaciones del tier Always Free:

  • 4 OCPUs Ampere A1 totales (distribuidos entre instancias)
  • 24 GB RAM total
  • 200 GB de almacenamiento Block Volume total
  • 1 Flexible Load Balancer (10 Mbps)
  • Outbound data transfer: 10 TB/mes

Compatibilidad de Shapes:

  • Ubuntu: Compatible con Ampere A1 (ARM) y x86
  • Windows Server: Solo x86 (VM.Standard3.Flex, VM.Standard.E4.Flex, etc.), Ampere A1 NO soporta Windows

Dominios:

  • Requiere dominio propio para DNS y certificados SSL
  • Nameservers deben apuntar a OCI DNS

Regiones OCI:

  • No todas las regiones tienen 3 Availability Domains
  • Always Free disponible en regiones de origen del tenancy únicamente

Seguridad:

  • Cambiar bastion_allowed_cidrs de 0.0.0.0/0 a IPs específicas en producción
  • Rotar API keys periódicamente
  • Revisar logs de WAF regularmente

Confiabilidad

Alta disponibilidad:

  • Distribución en 3 Availability Domains
  • Load Balancer con health checks cada 10 segundos
  • Autoscaling reemplaza instancias unhealthy automáticamente
  • Failover automático entre ADs

Recuperación ante fallos:

  • RTO (Recovery Time Objective): Si una instancia falla autoscaling crea nueva instancia en <5 minutos
  • AD completo falla: Tráfico redirigido a ADs restantes automáticamente
  • Load Balancer falla: OCI gestiona redundancia (SLA 99.99%)

Acuerdos de nivel de servicio (SLA) de la solución

Componente SLA mensual de OCI Downtime máximo/mes
Load Balancer 99.99% ~4 minutos
Compute Instances 99.95% (regiones con un solo AD) ~22 minutos
DNS 99.99% ~4 minutos

SLA compuesto de la solución:

  • Con 1 AD: 99.95% (~22 minutos downtime/mes)
  • Con 3 ADs: 99.99% (~4 minutos downtime/mes)

Importante: SLAs aplican a recursos pagos. Always Free no tiene SLA garantizado pero históricamente mantiene disponibilidad similar.

Alcance

Lo que se incluyó

  • VCN con subredes pública/privada
  • Load Balancer con listeners HTTP/HTTPS
  • WAF con protección OWASP, geo-blocking, rate limiting
  • Instance Pool con autoscaling basado en CPU
  • Soporte Ubuntu y Windows Server
  • DNS Zone con registros A para dominio
  • Importación de certificados SSL
  • Logging centralizado (LB, WAF, Audit)
  • Bastion Service para acceso seguro
  • Multi-AZ deployment (3 Availability Domains)
  • Infraestructura como código con Terraform
  • Seguridad en las capas de red, aplicación, transporte, acceso y datos

Lo que no está incluido

  • Object Storage para archivos estáticos y archivado de logs
  • Backup automatizado de datos de aplicación
  • CI/CD pipeline (GitHub Actions, Jenkins)
  • Monitoreo avanzado (Prometheus, Grafana)
  • Secrets management (Vault, OCI Secrets)
  • Multi-región deployment

Valor y resultados de la solución

Ahorro de tiempo:

  • Despliegue manual: 2-3 días → Terraform: <10 minutos (reproducible en múltiples ambientes)
  • Configuración WAF manual: 4-6 horas → Automatizado: <15 minutos
  • Escalado manual de instancias: 15-20 minutos → Autoscaling: 3-5 minutos
  • Aprovisionamiento de nueva instancia: 45-60 minutos → Instance Pool: 5-8 minutos

Ahorro de costos:

  • Ahorro total estimado del 20-60% mensual gracias a autoscaling vs. infraestructura estática
  • Infraestructura comparable en AWS: ~$150/mes → OCI Always Free: $0/mes
  • WAF comercial (CloudFlare Pro): $20/mes → OCI WAF: $0/mes

Reducción de riesgos:

  • Infraestructura 100% como código, idempotente y auditable
  • Protección OWASP Top 10, geo-blocking y rate limiting desde el día 1
  • Testing pre-deployment detecta problemas antes de aplicar cambios

Escalabilidad y resiliencia:

  • Autoscaling responde a picos de tráfico en <5 minutos, sin intervención manual
  • Arquitectura multi-AZ con failover automático entre zonas

Mantenibilidad:

  • Código modular reutilizable entre ambientes
  • Documentación como código y versionado en Git con rollback rápido

Mejoras y evolución futura

  • Implementar Object Storage para archivos estáticos y archivado de logs
  • Añadir alertas automáticas (email/notificaciones) en eventos críticos
  • Implementar CI/CD con GitHub Actions
  • Implementar blue-green deployment
  • Implementar multi-región para disaster recovery
  • Añadir serverless functions (OCI Functions) para tareas asíncronas
  • Configurar testing de integración post-deployment
